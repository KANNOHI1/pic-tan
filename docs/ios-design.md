# iOS UI デザイン原則（Pic-tan）

> SwiftUI / UIKit でUI を生成・修正するときはこの原則に従うこと。

## 設計原則

1. **Layered** — 背景 → サーフェス → テキスト/オブジェクト の3層で構成する
2. **Contrast** — テキストは背景に対してWCAG 2.1準拠（4.5:1以上）
3. **Minimal** — 1画面に使う色は3色まで（背景・アクセント・テキスト）
4. **Content First** — UIは黒子。コンテンツが主役で装飾は最小限に
5. **Calm Confidence** — 過剰なアニメーション・エフェクトより、正確なスペーシングと余白で品質を示す

## 禁止パターン

| 禁止 | 理由 |
|------|------|
| 強いドロップシャドウ | 「いかにもAI感」の典型。subtle shadowのみ許可 |
| カード上部・左端のカラーバー | AI生成UIの典型パターン。全周ボーダーで代替 |
| 派手なグラデーション・ネオンカラー | セマンティックカラー（Primary/Secondary）のみ |
| 絵文字の多用 | SF Symbolsとテキストで表現 |
| 色だけで情報伝達 | アイコン/テキストを必ず併用（アクセシビリティ） |
| 過剰なアニメーション | フィードバック目的（150〜300ms）のみ許可 |

## ブランドトーン

- **3語**: 静謐・精緻・温もり
- **参考**: Linear / Notion のようなミニマルで高速なSaaS感覚
- **アンチ**: Bootstrap的な没個性テンプレート、派手なグラデーションのアプリ
