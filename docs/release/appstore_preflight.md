## ステップ 1. アプリ内メタデータ（文言）の最終チェック
`docs/marketing/appstore_review_checklist.md` と `docs/release/appstore_submission_pack.md` および 実スクリーンショット を突き合わせ、以下に違反がないか確認します。

### 🚨 禁止表現（Forbidden）が含まれていないかチェック
「App Name」「Subtitle」「Promotional Text」「Description」「スクリーンショット」のすべてが対象です。

- [ ] **絶対的保証・過度な断定なし**（「必ず覚えられる」「バイリンガルになれる」「絶対に」「成績が上がる」等がないこと）
- [ ] **誇大解釈の不使用**（「IQが上がる」「右脳活性」「天才になる」等がないこと）
- [ ] **他社比較の不使用**（「長時間の学習は無意味」「他アプリより優れている」等の名指しや過激な下げがないこと）
- [ ] **医療アプローチの不使用**（「ADHDに効果がある」「発達障害の治療に」等がないこと）
- [ ] **不適切な親の免責表現なし**（「親は何もしなくていい」「スマホに子守りをただ任せる」等がないこと）

### ✅ 推奨・許可表現（Allowed）が遵守されているかチェック
- [ ] 学習の「プロセス支援」（記憶に残りやすい、サポートする等）に留まっているか
- [ ] プロダクトの「安心性」（広告なし等）が正当に主張されているか

---

## ステップ 2. 年齢レーティング（Age Rating）と整合しているか
Pic-tanのターゲット年齢（4〜8歳）およびApp Store上の子供向けカテゴリ（Kids Category）に合致する設定の確認です。

- [ ] App Store Connect の**年齢レーティング**設定が適切か（「Made for Kids」対象として 0〜5歳 または 6〜8歳枠 等のカテゴリ設定）
- [ ] アプリ内に暴力、恐怖、およびカテゴリ不適合な要素（外部ウェブへの無制限アクセスなど）が含まれていないか
- [ ] 親向けゲート（Parental Gate: 保護者のみが設定等にアクセスできる「九九を解かせる」等のハードル）は必要箇所に搭載されているか（※PT-017等での仕様確認）

---

## ステップ 3. プライバシーおよびトラッキング（Privacy / Tracking）
COPPA（米国児童オンラインプライバシー保護法）およびAppleの児童向けアプリ制限に関する最終確認です。

- [ ] 「広告なし」「トラッキングなし」を標榜している以上、SDK（解析・広告用）等の不要なトラッキングツールが意図せず混入していないか
- [ ] App Privacy（プライバシーラベル）の内容は「Data Not Collected（データ収集なし）」に設定してあるか、あるいは取得している情報（匿名化されたクラッシュログ等のみ）と完全に整合しているか
- [ ] アプリ内から外部リンク（プライバシーポリシー等以外）へ遷移する場合、Parental Gateを通しているか
