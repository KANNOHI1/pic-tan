## 📸 スクリーンショット構成と役割

Pic-tanのスクリーンショット（全5枚想定）は、ユーザーのインテンション（インストール意欲）を最大化するため、以下のような「ストーリー構成」で配置します。

| スライド番号 | 役割・目的 | 描画するUI要素の例 | A/Bテスト対象か |
| :---: | :--- | :--- | :---: |
| **1枚目** | **最重要フック（Hero）**<br>検索結果で最初に目に入る最大のアピール。アプリの核となる価値を1枚で伝える。 | プレイ画面全体と大きな可愛いキャラクター、目を引くトラストバッジ。 | ⭕️ 対象（最優先） |
| **2枚目** | **テーマ・価値の補完**<br>1枚目で興味を持った親に対し、アプリをダウンロードする最後の一押し（理由）を提供する。 | 保護者向けパネルや、オフライン環境、進捗画面などの周辺機能。 | ⭕️ 対象（最優先） |
| **3枚目** | **コア機能の説明**<br>どうやって遊ぶ（学ぶ）のかの3ステップを図解。 | 絵文字の拡大、解答の英語・日本語テキスト（3段階表示機能など）。 | ❌ 共通 |
| **4枚目** | **学習サイクルの説明**<br>ただの絵本ではなく「定着」する理由を説明。 | 自己評価ボタン（かんぺき / むずかしい）やリキューのUI。 | ❌ 共通 |
| **5枚目** | **トラスト保証**<br>親の信頼を決定づける安全性の念押し。 | 設定画面やプライバシーポリシーへのリンク、保護者向けインテント画面。 | ⭕️ 対象要素あり |

## 🧪 A/Bテスト対応表（デザイン原案とHTMLモックの紐付け）

現在、マーケティングの最優先検証事項として、「安心訴求」と「学習効果訴求」のどちらが親に響くかの A/B テストを実施します。各デザインは以下のリソースを参照して制作してください。

### 【パターンA】安心・環境訴求
スマホを渡す罪悪感を減らし、広告から子供を守ることを強調。

- **1枚目コピー**: 「完全広告なし！安心の知育アプリ」
- **2枚目コピー**: 「スマホを渡しても、罪悪感なし。」
- **モックアップ上の位置**: `screenshot_mockup.html` の 左列 (`.s1-a`, `.s2-a`)
- **参照ドラフト**: `docs/marketing/screenshot_copy_drafts.md` の「パターンA」

### 【パターンB】学習効果・習慣訴求
「絵で覚える」アプローチと、細かい反復学習（3分習慣）による定着を強調。

- **1枚目コピー**: 「絵で覚えるから、記憶にのこる。」
- **2枚目コピー**: 「1日たった3分。だから続く。」
- **モックアップ上の位置**: `screenshot_mockup.html` の 右列 (`.s1-b`, `.s2-b`)
- **参照ドラフト**: `docs/marketing/screenshot_copy_drafts.md` の「パターンB」

## ⚙️ 運用・差し替えフロー

1. 初期リリース時には、マーケティングチームの判断により、いずれかのパターン（または両方を用意して Custom Product Pages 等で出し分け）を App Store Connect へ登録します。
2. その後のマイナーアップデートや検証フェーズにおいて、効果が低かった場合のバリアント差し替えは「**1枚目と2枚目のペア**」で行うことを原則とします。
3. 3〜4枚目の共通UIスクショについては、アプリ内のUI言語変更（JP/USのローカライズ）時以外は過度なデザイン変更は行いません。
