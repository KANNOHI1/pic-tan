# Pic-tan デバイス QA テスト計画

**対象**: App Store 申請前 Gate 3 検証
**作成**: 2026-03-20 (PIC-7)
**実施環境**: Mac + Xcode シミュレーター & 実機
**前提**: PT-011（初回 simulator ビルド）通過済み

---

## 実施手順（共通）

1. `open apps/ios/PicTan.xcodeproj` で Xcode を開く
2. 対象デバイスをシミュレーターリストから選択
3. `Cmd+R` でビルド＆起動
4. 各チェック項目を上から順に実施し、結果を ✅ / ❌ / ⚠️ で記録する
5. ❌ が出たら再現手順を `tasks/HANDOFF_LOG.md` に追記してブロッカーとして報告する

---

## 1. 画面サイズ互換性

シミュレーターを以下の順に切り替えて全セクションのレイアウトを目視確認する。

| デバイス | 画面 | 確認ポイント |
|---|---|---|
| iPhone SE (3rd gen) | 4.7" 375×667pt | テーマカードが縦に収まるか / ボタンが切れないか |
| iPhone 14 | 6.1" 390×844pt | 標準レイアウト基準 |
| iPhone 14 Plus | 6.7" 428×926pt | 余白が過大にならないか |
| iPhone 15 Pro | 6.1" Dynamic Island | ノッチ回避・Hero 部が隠れないか |
| iPhone 15 Pro Max | 6.7" Dynamic Island | 大画面でカードが間延びしないか |
| iPad Air (11") | — | 対象外（iPhone only）— `INFO.plist` の `UIDeviceFamily` が `[1]` のみであること |

### チェックリスト

- [ ] SE: ホーム画面が 1 スクロールで全テーマカード表示される
- [ ] SE: 「スタート →」ボタンが 44pt 以上のタップ領域を持つ（目測）
- [ ] 全サイズ: カード学習画面で絵文字が画面の約 50% を占める
- [ ] 全サイズ: セッション完了画面のボタン 2 本が画面外に出ない
- [ ] Dynamic Island 機種: ヒーローセクションが Island と重ならない

## 2. Dynamic Type（アクセシビリティ）

設定アプリ → アクセシビリティ → 画面表示とテキストサイズ → さらに大きな文字

| 設定値 | 確認内容 |
|---|---|
| デフォルト（Medium） | 基準確認 |
| AX5（最大） | テキストが切れない・ボタンラベルが完全に表示される |
| AX1（小） | 小さすぎて読めない文字がないか |

### チェックリスト

- [ ] テーマカード名が AX5 でも 1 行内に収まるか、または適切に折り返されるか
- [ ] 「こたえは？」ヒントが AX5 で表示される
- [ ] 評価ボタン（完了時）が AX5 でも 2 ボタン横並びで表示される
- [ ] 保護者パネルのテキストが AX5 で読み切れる（スクロール可）

**VoiceOver（最低限確認）:**

- [ ] 設定 → アクセシビリティ → VoiceOver をオンにしてホーム画面を開く
- [ ] テーマカードをタップしたとき VoiceOver が読み上げる（英語名 or 日本語名）
- [ ] VoiceOver をオフに戻すことを忘れない

## 3. オフライン動作

機内モードにした状態で全フローを実施する。

**手順**: 設定 → 機内モード ON → アプリを完全終了 → 再起動

### チェックリスト

- [ ] ホーム画面が正常に表示される（クラッシュしない）
- [ ] テーマカード「どうぶつ」を選択 → カード学習が開始される（バンドル JSON から読み込み）
- [ ] カードに絵文字が表示される（PNG 読み込み失敗時のフォールバック）
- [ ] 8 枚完走 → セッション完了画面が表示される
- [ ] 連続学習日数（streak）が更新される
- [ ] 機内モード OFF → アプリを再起動 → streak・まちデータが保持されている

## 4. 年齢帯切り替え（Age-band）

### 前提確認

`packages/content/data/ja-JP/animals.json` の各カードに `difficulty` フィールドが設定されていること。

### テスト手順

1. 「保護者の方へ」をタップ → 計算問題に正解してパネルを開く
2. 「2〜3歳」を選択 → 閉じる → 「どうぶつ」を開始
3. 「4〜5歳」に変更 → 同様に開始
4. 「6歳以上」に変更 → 同様に開始

### チェックリスト

- [ ] 2〜3歳: difficulty=1 のカードのみが出る（JSON で最大 difficulty を確認）
- [ ] 4〜5歳: difficulty 1〜2 のカードが出る
- [ ] 6歳以上: すべてのカードが出る
- [ ] 設定変更後のカード枚数が変わることを確認（SE で確認しやすい）
- [ ] 年齢帯設定がアプリ再起動後も保持される

## 5. 保護者パネルフロー

### 5a. 親ゲート（Trust Gate）

- [ ] 「保護者の方へ ▾」をタップ → 計算問題が表示される
- [ ] 誤答 → 「もう一度どうぞ」が表示され、新しい問題が出る
- [ ] 正答 → パネルが開く
- [ ] パネルが開いた状態で再タップ → パネルが閉じる
- [ ] 子ども向け画面に数字キーパッドが表示される（テキストフィールドにキーボード型 `.numberPad` が適用）

### 5b. パネル内容

**セッション前（学習記録なし）:**

- [ ] streak バナーが非表示（count = 0 の場合）
- [ ] まちの成長セクションが非表示（住民 0 の場合）
- [ ] 「今日の学習記録」セクションが非表示
- [ ] 年齢帯ピッカーが表示される
- [ ] 安全バッジ 5 本が表示される

**1セッション完了後:**

- [ ] 「今日の学習記録」に テーマ名・正解数 が表示される
- [ ] スペースドリペティション説明文が表示される

**streak = 2 日以上（UserDefaults を手動編集するか、2 日連続で使用）:**

- [ ] streak バナー（✨/🌟/🔥）がホーム画面上部に表示される
- [ ] パネル内に streak カードが表示される

## 6. まち成長エッジケース

以下はシミュレーターの UserDefaults を直接編集して検証する。

**Xcode → Devices & Simulators → アプリ Container を開く → Library/Preferences/**.plist を編集

または `xcrun simctl` を使用:

```bash
xcrun simctl get_app_container booted <Bundle-ID> data
# → ~/Library/Developer/CoreSimulator/... へのパスが表示される
```

### チェックリスト

**レベル上限:**

- [ ] `pictan_residents` に 12 体以上の住民データを入れる → `townState.level == 12`（13 にならない）
- [ ] パネルで「レベル 12 ／ 住民 12 匹」と表示される
- [ ] 12 体目以降の住民が追加されても level が 12 を超えない

**リワード解放:**

- [ ] 住民 3 体 → 「town_reward_built_tree」のロック解除（パネルに未表示でも crash がない）
- [ ] 住民 6, 9, 12 体での各リワード（クラッシュなし、将来 UI 実装の準備として確認）

**ゼロ住民（初回起動模擬）:**

- [ ] UserDefaults を削除（アプリを削除 → 再インストール）→ まちセクションが非表示

## 7. パフォーマンス（3 分セッション目標）

### 7a. 起動時間

- [ ] コールド起動（最初の起動）から HomeView 表示まで **3 秒以内**
- [ ] テーマタップ → CardStudyView 表示まで **1 秒以内**

### 7b. セッション完走タイム

ストップウォッチで測定:

- [ ] 8 枚カード（2 タップ + 評価 × 8）= 約 2〜4 分が目安
- [ ] 3 分以内に完走できることを確認（ゆっくりタップ操作で）

### 7c. アニメーション

- [ ] カードフリップがスムーズ（60fps / 遅延なし）
- [ ] セッション完了の星アニメーションが詰まらない
- [ ] ヒーローの絵文字切り替えが 2.2 秒ごとに発生する

### 7d. メモリ

Xcode の Instruments → Allocations を起動して確認（任意）:

- [ ] 3 セッション連続後でもメモリが 100MB 以下（目安）
- [ ] セッション完了 → ホームへ戻る → 再度セッションでリーク未発生

## 8. Kids App 申請要件チェック

Apple の Made for Kids ガイドラインへの準拠確認。

| 要件 | 確認方法 | 結果 |
|---|---|---|
| 広告 SDK なし | `Package.swift` / `project.pbxproj` に広告 framework なし | - |

## Appendix A: シミュレーター UserDefaults 編集コマンド

```bash
# アプリの Container パスを取得
APP_CONTAINER=$(xcrun simctl get_app_container booted <bundle_id> data)

# streak を 7 日に設定
defaults write "$APP_CONTAINER/Library/Preferences/<bundle_id>.plist" \
  pictan_streak_count -int 7

# 全データ削除（初回起動模擬）
xcrun simctl uninstall booted <bundle_id>
xcrun simctl install booted <path_to_app>
```

`<bundle_id>` は Xcode → Target → General → Bundle Identifier で確認。
